# 🚀 Пошаговое руководство: От AI Studio до работающего сайта (GitHub + Render)

Это руководство написано специально для тех, кто делает это впервые. Мы пройдем путь от регистрации до готовой ссылки на ваш сайт.

---

## Часть 1: Подготовка и экспорт на GitHub

### 1. Создание аккаунта и репозитория
1.  Зайдите на [github.com](https://github.com) и зарегистрируйтесь.
2.  На главной странице нажмите зеленую кнопку **New** (слева) или иконку **+** (справа сверху) -> **New repository**.
3.  **Настройки репозитория:**
    *   **Repository name:** Введите название, например `my-build-sync`.
    *   **Public/Private:** Выберите *Private* (если не хотите, чтобы другие видели код) или *Public*.
    *   **НЕ ставьте галочки** на "Add a README", "Add .gitignore" (они уже есть в проекте).
    *   Нажмите **Create repository**.

### 2. Экспорт из Google AI Studio
1.  В окне AI Studio найдите иконку **Settings** (шестеренка в левом нижнем углу) или меню сверху.
2.  Выберите пункт **"Export to GitHub"**.
3.  Если появится окно авторизации — разрешите доступ к вашему GitHub.
4.  В списке выберите свой свежесозданный репозиторий (`my-build-sync`).
5.  Нажмите **Export**. Через 10-20 секунд ваш код появится на GitHub. Обновите страницу GitHub, чтобы увидеть файлы.

## Часть 2: Настройка базы данных на Render

Чтобы сайт сохранял данные, нам нужна база данных. Render дает её бесплатно.

1.  Зарегистрируйтесь на [render.com](https://render.com) (лучше через кнопку "Sign in with GitHub").
2.  Нажмите **New +** -> **PostgreSQL**.
3.  **Настройки БД:**
    *   **Name:** `crm-db`
    *   **Region:** Выберите любой ближайший (например, Frankfurt).
    *   **Plan:** Выберите **Free**.
4.  Нажмите **Create Database**.
5.  **ВАЖНО:** Дождитесь статуса "Available". Внизу страницы найдите блок **Connections**. Вам нужна строка **Internal Database URL** (она начинается на `postgres://...`). Скопируйте её.

---

## Часть 3: Запуск сайта (Web Service) на Render

Теперь превратим код в работающий сайт.

1.  Нажмите **New +** -> **Web Service**.
2.  Выберите вариант **"Build and deploy from a Git repository"**.
3.  В списке выберите ваш репозиторий `my-build-sync` (нажмите **Connect**).
4.  **Главные настройки (заполняйте внимательно):**
    *   **Name:** любое имя для ссылки сайта (например, `rogosin-portal`).
    *   **Language:** `Node`
    *   **Build Command:** `npm install && npm run build`
    *   **Start Command:** `npm run start` (это запустит наш `server.ts`).
5.  **Тонкая настройка (Environment Variables):**
    *   Нажмите кнопку **Advanced** -> **Add Environment Variable**.
    *   **Ключ 1:** `DATABASE_URL`
    *   **Значение 1:** Вставьте ту самую ссылку `postgres://...`, которую скопировали на шаге 2.
    *   **Ключ 2:** `GEMINI_API_KEY`
    *   **Значение 2:** Вставьте ваш ключ ИИ (если планируете использовать помощника).
6.  Нажмите **Create Web Service**.

---

## Часть 4: Как понять, что всё получилось?

1.  В окне Render вы увидите процесс «сборки» (Logs). Это занимает 2-3 минуты.
2.  Когда появится надпись **"Your service is live"**, в верхней части экрана появится ссылка (например, `https://rogosin-portal.onrender.com`).
3.  Перейдите по ней. Теперь ваш портал работает в интернете!

## 🛠 FAQ для новичка:
*   **Где мой пароль от БД?** Он зашит в ссылке `DATABASE_URL`. Никому её не показывайте.
*   **Как обновить сайт?** Просто измените код в AI Studio и экспортируйте снова, или исправьте файл прямо на GitHub. Render сам увидит изменения и обновит сайт через минуту.
*   **Сайт «засыпает»?** На бесплатном тарифе Render сайт уходит в спящий режим, если его никто не посещает. Первая загрузка после долгого перерыва может занять 30-50 секунд — это нормально.
