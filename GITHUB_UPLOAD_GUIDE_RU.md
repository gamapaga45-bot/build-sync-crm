# 🚀 Руководство по загрузке BuildSync CRM на GitHub (Без использования терминала)

Если у вас нет доступа к командной строке (Shell) или вы предпочитаете графический интерфейс, следуйте этой инструкции для прямой загрузки через браузер.

## 📦 Шаг 1: Получение кода из AI Studio

1.  В **AI Studio Build** в верхнем меню (обычно иконка настроек или экспорта) выберите **"Download as ZIP"** (Скачать как ZIP-архив).
2.  Распакуйте архив в удобную папку на вашем компьютере.
3.  **Важно:** Убедитесь, что внутри распакованной папки есть файлы `package.json`, `.gitignore` и папка `src`.

## 🔨 Шаг 2: Создание репозитория на GitHub

1.  Зайдите на [github.com](https://github.com) и войдите в аккаунт.
2.  Нажмите зеленую кнопку **New** (или иконку "+" -> New repository).
3.  **Repository name**: введите `buildsync-crm`.
4.  Выбирайте **Private** (Рекомендуется), чтобы ваш код и конфигурации не были публичными.
5.  **НЕ** ставьте никакие галочки (README, gitignore и т.д.).
6.  Нажмите **Create repository**.

## 📤 Шаг 3: Прямая загрузка файлов через браузер

1.  На странице вашего нового (пустого) репозитория найдите ссылку **"uploading an existing file"** (она находится в тексте под заголовком "Quick setup").
2.  Откройте папку с распакованным кодом на вашем компьютере.
3.  **Выделите ВСЕ файлы и папки** (кроме папки `node_modules`, если она вдруг там есть) и перетащите их прямо в окно браузера на GitHub.
4.  Дождитесь завершения загрузки всех файлов (их может быть много).
5.  Внизу в поле "Commit changes" напишите: `Initial upload: BuildSync CRM`.
6.  Нажмите зеленую кнопку **Commit changes**.

## ☁️ Шаг 4: Настройка ключей (Secrets)

После того как файлы появились на GitHub, настройте переменные окружения, чтобы приложение могло работать в облаке:
1.  Перейдите во вкладку **Settings** вашего репозитория.
2.  В меню слева выберите **Secrets and variables** -> **Actions**.
3.  Нажмите **New repository secret**.
4.  Добавьте следующие переменные:
    -   `GEMINI_API_KEY` — ваш ключ от Google Gemini.
    -   `CLOUDFLARE_ACCOUNT_ID` — ID аккаунта Cloudflare.
    -   `R2_ACCESS_KEY_ID` — Ключ доступа R2.
    -   `R2_SECRET_ACCESS_KEY` — Секретный ключ R2.
    -   `R2_BUCKET_NAME` — Название бакета.

## 🏗 Как запустить проект в интернете (Деплой)

Теперь, когда код на GitHub, вы можете подключить его к хостингу:
1. Зарегистрируйтесь на [Render.com](https://render.com).
2. Нажмите **New** -> **Web Service**.
3. Подключите свой GitHub аккаунт и выберите репозиторий `buildsync-crm`.
4. Render сам предложит настройки:
   - Build Command: `npm run build`
   - Start Command: `node server.ts`
5. В разделе **Environment Variables** добавьте те же ключи, что вы сохранили в Secrets на GitHub.
